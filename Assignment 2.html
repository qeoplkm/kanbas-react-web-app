1 Introduction

A polynomial is made of several terms, each term having a coefficient and a variable raised to a power. Polynomials are one-variable if all their terms contain only one variable. An example of such a polynomial is f(x)=3x45x3+2x4 . This polynomial has four terms.

The degree of a polynomial is defined as the highest power of the variable in a term. In the above example, the degree of the polynomial is 4. The polynomials we deal with have only non-negative whole numbers as powers. The coefficients of our polynomials are integral numbers.

Two polynomials are the same if they contain the same terms.

Several algebraic operations are possible with polynomials. The simplest one is evaluating the polynomial for a specific value of the variable. For example, for a polynomial f(x)=3x45x3+2x4 , its value at x=2.5 is defined as f(2.5)=3(2.5)45(2.5)3+2(2.5)4 which is 40.0625 .

Polynomials can be added together to create a new polynomial. The addition is performed by combining all the terms and adding the coefficients of the terms with the same power. For example (3x45x3+2x4) + (2x3+2x2+4) = 3x43x3+2x2+2x . The degree of the sum is the maximum of the degrees of the two polynomials.

Polynomials can be multiplied together to create a new polynomial. To multiply two polynomials f(x) and g(x) we multiply g(x) with each term in f(x) to produce new polynomials and then add them together. For example 3x45x3+2x4 * 2x3+2x2+4 is 6x7+6x6+12x410x610x520x3+4x4+4x3+8x8x38x216 which reduces to 6x74x610x5+16x424x38x2+8x16 . The degree of the product is equal to the sum of the degrees of the two polynomials.

The derivative of a polynomial (from differential calculus) is a polynomial made up of the derivative of each term. The derivative of a term axb is a term with coefficient ab with the variable raised to the power b1 . The derivative of a constant term ( x0 ) is 0 . For example the derivative of 3x45x3+2x4 is 12x315x2+2 .

2 What to do

All your interfaces and classes should be in the polynomial package. The tests should be in the default package.

2.1 Interface

A Polynomial interface has been provided to you, that defines the operations that must be supported by an implementation of a polynomial. You are not allowed to change this interface in any way .

2.2 Implementation

Now implement this interface in a class SimplePolynomial . Beyond implementing the Polynomial interface, this implementation should have the following features/obey these constraints:

  • This representation must use an array-like data structure to explicitly store all possible terms of the polynomial. For example, a polynomial with degree 4 should have four distinct terms stored in this representation. The order of these terms is up to you.

  • This class should have a constructor with no parameters that creates a polynomial with no terms, i.e. the polynomial 0 .

  • This class must provide a standard way to determine if two polynomials are the same.

  • You are not allowed to implement the add , multiply or derivative methods using string manipulations. You are expected to implement the arithmetic suitably.

  • This class should include a toString method that returns a string that contains the polynomial. The following examples should help you infer the required format:

    • 5x2+4x2 creates the string 5x^2+4x^1-2

    • 50x3+x2+3 creates the string -50x^3+1x^2+3

    • 4x+2x53x210 creates the string 2x^5-3x^2+4x^1-10

    • The empty polynomial should create the string 0

Write tests that thoroughly test this implementation. As always, it is recommended to write the test before completing the SimplePolynomial implementation.

3 What to submit

Please submit all your source files and tests in a single zip file (including the code that was provided to you).

4 Criteria for grading

You will be graded on:

  1. Whether your implementation obeys all the above constraints.

  2. The design of your implementation.

  3. Whether you have placed appropriate restrictions on attributes and methods.

  4. The correctness of your methods.

  5. The modularity of your implementation.

  6. The quality and coverage of tests.

  7. The quality of documentation.

  8. The style of your code.
